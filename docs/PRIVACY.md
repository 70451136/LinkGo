# 隐私与本机使用边界

LinkGo 设计上更偏向本机优先工作流，但实际隐私边界取决于你的模型和部署方式。

## 本机保留

- 选择本机目录作为沙箱时，目录不应上传到云端服务器。
- 本机沙箱和 LinkClaw 控制能力只允许本机访问。
- 结果文件优先写入本机输出目录。

## 可能联网的情况

- 使用在线大模型。
- 开启联网搜索。
- 下载或更新运行时依赖。
- 使用外部 OCR、语音识别、图片生成等服务。

## 使用建议

- 处理敏感文件时优先选择本地模型。
- 不要公开分享含 API Key、Token、Cookie、客户数据的日志。
- 发布 Issue 前请移除敏感内容。
- 局域网或公网部署时，不要开放本机沙箱控制接口。
